## Définition du DCA

Le DCA (Dollar-Cost Averaging) consiste à investir un montant fixe à un rythme fixe (hebdo/mensuel).

Intérêt principal : le comportement.

- moins de pression sur le “bon timing”
- routine simple
- régularité

Mais le DCA ne supprime pas le risque : si le marché baisse, la valeur peut baisser.

## Pour qui ?

Le DCA est souvent adapté si vous :

- investissez long terme
- voulez réduire l’émotion
- préférez un process à une prédiction

Moins adapté si vous :

- avez besoin de l’argent à court terme
- ne supportez pas les drawdowns
- changez de stratégie toutes les semaines

## Mettre en place une politique

Écrivez une politique tenue 3 mois :

1. **Montant** : fixe, supportable.
2. **Rythme** : hebdo ou mensuel.
3. **Liste d’actifs** : petite et comprise.
4. **Allocation max** : limite globale crypto dans votre plan.
5. **Cadence de revue** : mensuelle/trimestrielle (pas quotidienne).

## Exemple de politique DCA (illustratif)

L’idée est d’écrire une fiche courte que vous suivez 90 jours sans la modifier chaque semaine :

| Élément | Votre règle |
|---|---|
| Montant | ____ (supportable sans stress) |
| Rythme | hebdo / mensuel |
| Liste d’actifs | ____ (petite et comprise) |
| Allocation max crypto | ____ (limite globale) |
| Revue | mensuelle / trimestrielle |
| Règle d’ajustement | “si allocation > max → je ralentis/je stoppe” |

Le bénéfice principal du DCA, c’est la **discipline**. Si vous changez la règle chaque semaine, vous perdez l’avantage.

## Frais et frictions

Avant d’automatiser, vérifiez :

- frais vs spreads (une partie est “dans le prix”)
- dépôt/retrait (méthode + réseau)
- conversion de devise (FX)
- limites/minimums

Des frais mal compris = fuite lente.

## Sécurité

La sécurité fait partie du plan :

- gestionnaire de mots de passe + mot de passe unique
- 2FA + codes de secours
- email sécurisé

Comprenez aussi la custody : selon la plateforme, vous ne contrôlez pas directement les clés. C’est pratique, mais ça introduit un risque de contrepartie.

## Erreurs fréquentes

- **Pas d’allocation max :** le DCA grossit sans contrôle.
- **Ignorer les frais :** petits coûts × beaucoup d’achats.
- **Changer de rythme :** on perd le bénéfice principal.
- **Levier :** volatilité → liquidation.
- **Pas d’historique :** cost basis et suivi impossibles.

## Checklist

- Montant supportable sans stress.
- Allocation max crypto + cadence de revue définies.
- Frais/spreads + règles dépôt/retrait compris.
- 2FA + email sécurisés.
- Historique exporté/sauvegardé régulièrement.
